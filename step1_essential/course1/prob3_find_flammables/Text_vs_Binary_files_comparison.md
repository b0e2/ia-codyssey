# 텍스트 파일(Text File)과 이진 파일(Binary File)의 비교 및 장단점 분석

## 1. 개념 및 핵심 차이점

데이터를 디스크에 저장하는 방식에 따라 크게 텍스트 파일과 이진 파일로 나뉩니다. 두 방식의 가장 큰 차이점은 **'사람이 읽을 수 있도록 문자로 인코딩(Encoding) 과정을 거치느냐, 아니면 컴퓨터 메모리의 형태(0과 1) 그대로 저장하느냐'**에 있습니다.

* **텍스트 파일 (Text File):** 모든 데이터를 문자열(String)로 변환하여 표준 문자 인코딩(ASCII, UTF-8 등) 규칙에 따라 저장하는 파일입니다.
* **이진 파일 (Binary File):** 컴퓨터가 데이터를 메모리에서 처리하는 원형의 바이트(Byte) 형태 그대로 저장하는 파일입니다. 

---

## 2. 장단점 비교

### 📝 텍스트 파일 (Text File)
* **대표 포맷:** `.csv`, `.txt`, `.json`, `.md`
* **장점 (Pros):**
    * **가독성:** 메모장이나 VS Code 등 일반적인 텍스트 에디터로 열어서 사람이 바로 내용을 확인하고 직관적으로 수정할 수 있습니다.
    * **호환성:** 운영체제(OS)나 플랫폼에 관계없이 데이터를 주고받거나 통합하기가 매우 쉽습니다.
    * **디버깅 용이:** 데이터의 형태를 눈으로 직접 볼 수 있어 오류를 찾기 수월합니다.
* **단점 (Cons):**
    * **비효율적인 공간 차지:** 데이터 타입과 무관하게 모든 것을 문자로 저장하므로 저장 공간을 더 많이 차지합니다. (예: 숫자 `12345`를 이진수로 저장하면 4바이트면 충분하지만, 텍스트로 저장하면 각 자리수마다 1바이트씩 총 5바이트가 필요합니다.)
    * **느린 처리 속도:** 읽고 쓸 때마다 문자를 실제 데이터 타입(int, float 등)으로 변환(Parsing)하는 과정이 필요하므로 오버헤드가 발생합니다.

### 💻 이진 파일 (Binary File)
* **대표 포맷:** `.bin`, `.dat`, `.jpg`, `.exe`
* **장점 (Pros):**
    * **처리 속도 최적화:** 메모리에 올려진 데이터 구조 그대로 저장하고 읽어오므로, 별도의 인코딩/디코딩이나 형변환 과정이 생략되어 처리 속도가 압도적으로 빠릅니다.
    * **저장 공간 절약:** 데이터를 가장 효율적인 크기로 압축하여 저장하므로 텍스트 파일에 비해 파일 용량이 훨씬 작습니다.
* **단점 (Cons):**
    * **가독성 불가:** 사람이 텍스트 에디터로 열면 글자가 깨진 형태(예: `*`)로 보이며 직접 읽거나 수정할 수 없습니다.
    * **종속성:** 파일에 저장된 데이터의 정확한 구조(스키마)와 읽어들이는 프로그램의 로직이 없으면 데이터를 해석할 수 없습니다.

---

## 3. 요약 및 실무 적용
* 데이터의 **호환성, 디버깅, 사람이 직접 읽고 편집해야 하는 목적**이라면 **텍스트 파일**을 사용합니다.
* 대용량 데이터의 **빠른 처리 속도, 저장 공간의 최적화, 보안**이 주 목적이라면 **이진 파일**을 사용합니다.
